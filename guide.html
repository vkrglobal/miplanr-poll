Three components work together:

📄 index.html — Poll page

A beautiful standalone page hosted on Netlify. Each poll is a unique URL with your question and options as parameters. No database needed for MVP.

⚡ parse-poll-result.js — AI parser

A Netlify serverless function powered by Claude. POST any WhatsApp text to it and receive a structured calendar event JSON ready to push to any platform.

📅 Calendar sync

Google Calendar and Outlook via deep-link URLs (instant, no auth). All 8 platforms via API calls using your stored credentials.

🤖 miPlanr 2.0 chat

The poll result and winning date feeds back into miPlanr's AI life coach, which can add it to your Life Plan and trigger follow-up actions.

💡WhatsApp limitation: WhatsApp has no public API for poll results. The parser solves this by letting you paste the winning result as text — Claude extracts the structured event automatically. For fully automatic sync, use the native miPlanr poll instead.

5-minute setup
Deploy the poll page to Netlify
▾
1
Create a new GitHub repository
Go to github.com/new. Name it miplanr-poll. Leave it public. Click Create repository. Then upload all the files from your unzipped folder using the Upload files button. Commit directly to main.
2
Connect to Netlify
Go to app.netlify.com → Add new site → Import an existing project → GitHub. Authorise Netlify to access your GitHub. Select the miplanr-poll repository.
3
Configure build settings
Netlify will auto-detect settings from netlify.toml. Confirm: Publish directory: . and Functions directory: netlify/functions. Click Deploy site.
4
Add your Anthropic API key
In Netlify: go to your site → Site configuration → Environment variables → Add a variable. Set key: ANTHROPIC_API_KEY. Paste your Claude API key as the value. Save. Then go to Deploys → Trigger deploy → Deploy site to apply the env var.
⚠️Never add the API key to your HTML or commit it to GitHub. Environment variables are encrypted by Netlify and never exposed publicly.
5
Your poll page is live!
Netlify gives you a URL like amazing-poll-123.netlify.app. You can customise this under Domain settings → Change site name to something like miplanr-poll.netlify.app. Visit it — you should see the miPlanr poll page.

Day-to-day use
How to create and share a poll
▾

Every poll is a URL with your details baked in as parameters. No database needed. Share the URL anywhere — WhatsApp, email, SMS.

1
Build your poll URL
Take your Netlify URL and add parameters. Replace spaces with + and separate options with |.
https://miplanr-poll.netlify.app/?
q=When+works+for+our+team+lunch?&
creator=Josh+Sim&
event=Team+Lunch&
deadline=30+Jun&
opts=Saturday+5+Jul+2pm|Sunday+6+Jul+12pm|Monday+7+Jul+1pm&
threshold=5
2
URL parameters explained
q — Your poll question
creator — Your name (shows on the poll)
event — What gets booked in the calendar when the poll closes
deadline — Display text for closing date
opts — Options separated by the | pipe character
threshold — How many votes needed before auto-confirming
3
Share the link
Paste the URL directly into a WhatsApp group, email, or SMS. When people tap it, they land on the branded miPlanr poll page. No app download needed.
💡For long URLs, use a URL shortener like bit.ly or tinyurl.com to make it shareable. Or set up a custom domain under Netlify domain settings.
4
Shortcut: ask miPlanr 2.0 to build the URL
In the miPlanr 2.0 chat, type: "Create a poll asking when works for our team meeting, options are Monday 7 July 10am, Tuesday 8 July 2pm, Wednesday 9 July 9am, closing this Friday, 5 votes needed." — miPlanr builds the URL for you instantly.

Calendar sync
Google Calendar integration
▾

Two levels: instant deep-link (no setup) and full API sync (requires OAuth).

A
Level 1 — Instant deep-link (already works, no setup)
When a voter clicks Google Calendar on the poll results page, it opens Google Calendar pre-filled with the event. The voter clicks Save. Done. This works today with zero configuration.
B
Level 2 — Full API sync (auto-books without voter action)
For automatic booking when the poll closes, you need a Google Cloud project and OAuth credentials.
Setup steps:
1
Go to console.cloud.google.com → Create project → name it miPlanr Poll
2
Enable Google Calendar API: APIs & Services → Enable APIs → search Calendar → Enable
3
Create credentials: APIs & Services → Credentials → Create → OAuth 2.0 Client ID → Web application. Add your Netlify URL as an authorised origin.
4
Download the credentials JSON. Add GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET to Netlify environment variables.
5
The parse-poll-result.js function includes the googleCalUrl in its response — use this to call the Calendar API server-side when the poll threshold is reached.
// Example API call in your Netlify function POST https://www.googleapis.com/calendar/v3/calendars/primary/events Authorization: Bearer {access_token} Content-Type: application/json { "summary": "Team Lunch — Saturday 5 Jul, 2pm", "start": { "dateTime": "2025-07-05T14:00:00", "timeZone": "Europe/London" }, "end": { "dateTime": "2025-07-05T15:00:00", "timeZone": "Europe/London" }, "description": "Scheduled via miPlanr Poll · miplanr.com" }

Outlook / Microsoft 365 integration
▾
A
Level 1 — Deep-link (already works)
The Outlook button on the poll page opens Outlook.com/Teams pre-filled. Works immediately with no setup.

Level 2 — Microsoft Graph API (full auto-sync)
Register an app in Azure Active Directory for automatic booking.
1
Go to portal.azure.com → Azure Active Directory → App registrations → New registration. Name it miPlanr Poll. Set redirect URI to your Netlify URL.
2
API permissions → Add → Microsoft Graph → Delegated → Calendars.ReadWrite. Grant admin consent.
3
Certificates & secrets → New client secret. Copy the value immediately. Add AZURE_CLIENT_ID, AZURE_CLIENT_SECRET, AZURE_TENANT_ID to Netlify env vars.
// Microsoft Graph API call POST https://graph.microsoft.com/v1.0/me/events Authorization: Bearer {access_token} Content-Type: application/json { "subject": "Team Lunch", "start": { "dateTime": "2025-07-05T14:00:00", "timeZone": "GMT Standard Time" }, "end": { "dateTime": "2025-07-05T15:00:00", "timeZone": "GMT Standard Time" }, "body": { "content": "Scheduled via miPlanr Poll", "contentType": "text" } }

AI parsing
WhatsApp poll result → miPlanr (Claude parser)
▾
ℹ️Why this exists: WhatsApp has no API. You cannot auto-pull poll results. This parser bridges the gap — you copy the winning option text from WhatsApp, paste it here, and Claude converts it to a structured calendar event.
1
Check your WhatsApp poll result
In your WhatsApp group, tap the poll to see results. The winning option is shown with the most votes. Long-press the winning option text and copy it. E.g.: "Saturday 5 July 2pm — 4 votes"
2
Send it to the miPlanr parser
Once your Netlify function is deployed, call it with a POST request. You can do this from miPlanr 2.0 chat, or directly:
POST https://your-site.netlify.app/api/parse-poll-result Content-Type: application/json { "rawText": "Saturday 5 July 2pm - Team lunch - 4 votes" }
3
Claude returns a structured event
The response includes everything you need to book the event:
{ "title": "Team Lunch", "date": "2025-07-05", "startTime": "14:00", "endTime": "15:00", "confidence": "high", "googleCalUrl": "https://calendar.google.com/...", "outlookDeeplink": "https://outlook.live.com/...", "platformPayloads": { /* ready for ChurchSuite, iSAMS, etc */ } }
4
Use the result to book across platforms
The platformPayloads object contains pre-built request bodies for every integration. Pass the relevant one to each platform's API (see sections 8–15 below).

Platform sync
ChurchSuite calendar sync
▾
1
Get your ChurchSuite API key
Log into ChurchSuite → My ChurchSuite → API → Generate API Token. Copy the token. Add it to Netlify env vars as CHURCHSUITE_API_KEY.
2
Find your organisation identifier
Your ChurchSuite URL looks like yourchurch.churchsuite.com. The subdomain (yourchurch) is your organisation identifier. Add as CHURCHSUITE_ORG.
3
Create the calendar event
Use the payload from the parser response:
POST https://api.churchsuite.com/v1/calendar/events X-Auth: {CHURCHSUITE_API_KEY} X-Account: {CHURCHSUITE_ORG} Content-Type: application/json { "name": "Team Lunch", "datetime_start": "2025-07-05T14:00:00", "datetime_end": "2025-07-05T15:00:00", "description": "Scheduled via miPlanr Poll", "visible_to": "everyone" }
💡Full ChurchSuite API docs: support.churchsuite.com → Developer → Calendar API.

Platform sync
iSAMS school calendar sync
▾
1
Enable the iSAMS API
Contact your iSAMS system administrator. They need to enable the REST API module and create an API client for miPlanr. You'll receive a Client ID and Client Secret.
2
Get an access token
iSAMS uses OAuth 2.0 client credentials flow:
POST https://[school].isams.cloud/auth/connect/token Content-Type: application/x-www-form-urlencoded grant_type=client_credentials &client_id={CLIENT_ID} &client_secret={CLIENT_SECRET} &scope=iSAMS
3
Create the calendar entry
POST https://[school].isams.cloud/api/calendar/events Authorization: Bearer {access_token} { "title": "Team Lunch", "startDate": "2025-07-05", "startTime": "14:00", "endTime": "15:00", "notes": "Scheduled via miPlanr Poll" }
💡Full docs: developerhub.isams.cloud. Replace [school] with your school's iSAMS subdomain.

Platform sync
Google Classroom sync
▾
ℹ️Google Classroom doesn't have a traditional calendar — miPlanr posts an Announcement to the relevant course, which appears in students' and teachers' feeds.
1
Enable Classroom API in Google Cloud
In your Google Cloud project (from Step 5), also enable the Google Classroom API. Add the scope classroom.announcements.create to your OAuth credentials.
2
Get the course ID
Go to classroom.google.com. Click your course. The URL contains the course ID: classroom.google.com/c/AbcDef123. Copy that ID.
3
Post the announcement
POST https://classroom.googleapis.com/v1/courses/{courseId}/announcements Authorization: Bearer {access_token} { "text": "📅 Team Lunch confirmed: Saturday 5 July at 2pm. Scheduled via miPlanr Poll.", "state": "PUBLISHED" }

Salesforce event sync
▾
1
Create a Connected App in Salesforce
Setup → App Manager → New Connected App. Enable OAuth. Add scopes: api and refresh_token. Set callback URL to your Netlify URL. Save — you'll get a Consumer Key and Consumer Secret.
2
Get an access token (username-password flow for server)
POST https://login.salesforce.com/services/oauth2/token Content-Type: application/x-www-form-urlencoded grant_type=password &client_id={CONSUMER_KEY} &client_secret={CONSUMER_SECRET} &username={SF_USERNAME} &password={SF_PASSWORD}{SF_SECURITY_TOKEN}
3
Create the Event object
POST https://[instance].salesforce.com/services/data/v58.0/sobjects/Event Authorization: Bearer {access_token} { "Subject": "Team Lunch", "ActivityDateTime": "2025-07-05T14:00:00.000Z", "DurationInMinutes": 60, "Description": "Scheduled via miPlanr Poll" }

POST https://api.hubapi.com/crm/v3/objects/meetings Authorization: Bearer {HUBSPOT_TOKEN} { "properties": { "hs_meeting_title": "Team Lunch", "hs_timestamp": 1751724000000, // unix ms "hs_meeting_end_time": 1751727600000, "hs_internal_meeting_notes": "Scheduled via miPlanr Poll" } }

miPlanr integration
Connect to miPlanr 2.0 chat
▾

The Poll Hub feeds back into miPlanr's AI life coach. Here's how to wire it together.

1
Add the poll URL endpoint to miPlanr 2.0's system prompt
In your miPlanr 2.0 Netlify function's system prompt, add the Poll Hub URL so the AI coach knows how to create polls:
// In miPlanr's system prompt "When a user asks to create a poll or schedule an event with a group, use the Poll Hub at https://miplanr-poll.netlify.app to generate a shareable URL. Format: https://miplanr-poll.netlify.app/?q={question}&opts={opt1}|{opt2}&event={title}"
2
Add the WhatsApp parser endpoint
Also tell miPlanr 2.0 about the parser so it can handle pasted WhatsApp results:
"When a user pastes a WhatsApp poll result, call: POST https://miplanr-poll.netlify.app/api/parse-poll-result with { rawText: [pasted text] } Then offer to book the returned event to their calendar."
3
Life Plan integration
When a poll result is booked, miPlanr's Relationships or Community coach should log it. Add to the AI prompt:
"After booking a poll event, ask the user which Life Plan domain this belongs to (Relationships, Community, Health, Career, Finance, Faith) and log it as a milestone in that domain's progress tracker."
4
Test the full flow
In miPlanr 2.0 chat, type: "Create a poll for our next HTB small group — when can everyone meet? Options: Tuesday 8 July 7pm, Wednesday 9 July 7pm, Thursday 10 July 7pm." — miPlanr should generate the poll URL, which you can share directly into the WhatsApp group.
💡This is the full Option C pipeline: WhatsApp = distribution only. miPlanr = poll engine + booking + life planning.

Help
Troubleshooting common issues
▾
?
Poll page shows blank / default content
Check your URL parameters are correctly encoded. Spaces must be + or %20. Options must be separated by | (pipe). Test your URL in a URL decoder like urldecoder.org.
?
Parser returns 502 error
Check that ANTHROPIC_API_KEY is correctly set in Netlify environment variables. Redeploy after adding — env vars don't apply to existing deploys. Check your Anthropic API key has credit.
?
Google Calendar button does nothing
The deep-link URL opens Google Calendar in a new tab. Check your browser isn't blocking pop-ups from your Netlify URL. Allow pop-ups for the site in browser settings.
?
Page won't embed in Google Sites
Make sure the _headers file is in the root of your repo and deployed. Check Netlify → Deploys → your latest deploy logs for any header errors. The X-Frame-Options: ALLOWALL header must be present.
?
Platform API calls returning 401
Your access token has expired. Most OAuth tokens expire after 1 hour. Implement token refresh logic in your Netlify function, storing the refresh token in an environment variable.
