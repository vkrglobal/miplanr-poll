ℹ️ Why not point nameservers to Netlify? Cloudflare registered the domain, so Cloudflare controls the nameservers. Changing them away would lose Cloudflare's free DDoS protection, analytics, and SSL features. Instead we keep DNS on Cloudflare and add records that point traffic to Netlify — best of both worlds.
1
Netlify
Add miplanr.com as a custom domain in Netlify
~3 min ▾
1
Open your Netlify site dashboard
Go to app.netlify.com and click on your miPlanr poll site (the one you deployed). In the left sidebar, click Domain management.
2
Add a custom domain
Click the Add a domain button. Type in: miplanr.com and press Enter. Netlify will check if you own it — click Add domain to confirm.
💡Also add www.miplanr.com as a second domain — Netlify will auto-redirect it to miplanr.com.
3
Get the DNS values Netlify needs
After adding, you'll see "Awaiting External DNS" next to both domains. Click the Check DNS configuration button on each. Netlify will show you the exact records to add — note them down. They look like this:
app.netlify.com / sites / miplanr-poll / domain-management
CNAME
www.miplanr.com
your-site.netlify.app
ALIAS
miplanr.com (apex)
apex-loadbalancer.netlify.com
STATUS
Awaiting External DNS
pending…
⚠️Copy your actual Netlify subdomain — it's unique to your site, like amazing-poll-abc123.netlify.app. Don't use a generic placeholder. Find it at the top of your site dashboard.
2
Cloudflare
Add DNS records in Cloudflare
~5 min ▾
1
Open Cloudflare DNS settings
Go to dash.cloudflare.com → click on miplanr.com → click DNS in the left sidebar → then Records.
2
Delete any existing A or CNAME records for @ and www
If there are any existing records for @ (root domain) or www, delete them first. They will conflict with the new records. Look for any records with type A or CNAME pointing to Cloudflare's parking page or anywhere else.
⚠️Do not delete MX records (email) or TXT records (verification). Only delete A and CNAME records for @ and www.
3
Add the CNAME record for www
Click Add record and fill in:
FieldValue
TypeCNAME
Namewww
Targetyour-site-name.netlify.app (your actual subdomain)
TTLAuto
Proxy status🔘 DNS only (grey cloud — see step 4)
💡Replace your-site-name.netlify.app with your actual Netlify subdomain from Phase 1 Step 3.
4
Add the CNAME record for the root domain (apex)
Cloudflare supports CNAME flattening — this lets you use a CNAME for the root domain (miplanr.com), which most DNS providers can't do. Click Add record again:
FieldValue
TypeCNAME
Name@ (the @ symbol = root domain)
Targetapex-loadbalancer.netlify.com
TTLAuto
Proxy status🔘 DNS only (grey cloud — see step below)
💡This is the key advantage of using Cloudflare — CNAME flattening means the root domain (miplanr.com without www) points to Netlify perfectly, which standard DNS can't do.
3
Cloudflare — Critical step
Turn OFF the orange proxy cloud
1 min ▾
🟠

This is the #1 cause of miplanr.com not working

Cloudflare has an "orange cloud" proxy that intercepts traffic before it reaches Netlify. When it's ON for these records, Netlify can't verify your domain or provision your SSL certificate. You must set both records to DNS only (grey cloud) for Netlify to work correctly.

1
Find the proxy toggle on each record
In Cloudflare DNS Records, for both your www CNAME and your @ CNAME, look at the Proxy status column. If it shows an orange cloud icon (🟠), click it to toggle it to a grey cloud (⚫ DNS only).
2
Confirm both records show grey cloud
Your DNS records table should look like this after the change:
dash.cloudflare.com → miplanr.com → DNS → Records
CNAME
www → your-site.netlify.app
⚫ DNS only
CNAME
@ → apex-loadbalancer.netlify.com
⚫ DNS only
💡Once Netlify's SSL certificate is active (usually within a few hours), you can turn the orange proxy back on — but many Netlify users leave it grey permanently and let Netlify handle SSL, which is simpler.
4
Netlify
Verify DNS and activate HTTPS
~2 min setup · then wait up to 24h ▾
1
Return to Netlify Domain management
Go back to app.netlify.com → your site → Domain management. After a few minutes (sometimes seconds), "Awaiting External DNS" should change to "DNS verified ✓".
2
Provision the SSL certificate
Scroll down to the HTTPS section. Click Verify DNS configuration. Netlify will request a free Let's Encrypt SSL certificate. This takes 1–15 minutes. Once done, you'll see "Your site has HTTPS enabled".
ℹ️If it says "Certificate provisioning" for more than 30 minutes, double-check the Cloudflare proxy is set to grey (DNS only) on both records.
3
Check DNS propagation

6
If things go wrong
Troubleshooting DNS issues
▾
?
Still seeing "DNS_PROBE_FINISHED_NXDOMAIN"
This means DNS hasn't propagated yet, or the records are wrong. Check: (1) Both CNAME records are saved in Cloudflare. (2) Proxy is grey/off on both. (3) Wait at least 30 minutes. (4) Try on mobile data (bypasses local DNS cache).
# Test in terminal (Mac/Linux) or Windows PowerShell: nslookup miplanr.com nslookup www.miplanr.com # Expected output should mention netlify.com www.miplanr.com CNAME your-site.netlify.app
?
SSL certificate stuck on "Provisioning"
This almost always means the Cloudflare orange proxy is ON. Go to Cloudflare → DNS → Records → find your CNAME records → click the orange cloud to make it grey. Then go back to Netlify → HTTPS → Verify DNS configuration.
?
www works but miplanr.com (root) doesn't
Make sure you added the @ CNAME pointing to apex-loadbalancer.netlify.com. Also check in Netlify that both miplanr.com and www.miplanr.com are added as domains (not just one). Netlify must know about both.
?
miplanr.com redirecting to the wrong site
In Netlify, go to Domain management and check which site the domain is assigned to. If you have multiple Netlify sites, the domain must be added to the correct one (the miplanr-poll site, not another project).
✉
Still stuck after 24 hours?
Contact Netlify Support Forums with a screenshot of your Cloudflare DNS records and your Netlify domain management screen. Or paste the error into miPlanr 2.0 chat for instant diagnosis.
